Stargazing App

The stargazing app is a cross-platform mobile and desktop application that aims to share the stories and legends told about the stars from the perspectives of many cultures, beyond the IAU's 88 modern constellations.
More

Stargazing App

The stargazing app is being developed in partnership with the University of Manitoba, as part of the STEAM (Science, Technology, Engineering, Art, and Math) initiative. The goal of the app is to preserve legends about the stars from around the world, and to share folklore that isn't as well-known as that of Greek and Roman mythology.

On mobile devices, the app can be controlled using touch input or with the device gyroscope, allowing the user to point their phone at the night sky to learn about what they see.

Users can click on any constellation to get more information and read stories about the constellation.

In the future, these pages will include more pictures and artwork, and may include recordings of storytellers sharing legends about the constellation, in an effort to preserve traditional oral storytelling.

The app fetches around 5000 stars from a database to display on a virtual celestial sphere. It draws lines between the stars to show constellations for the currently selected culture, and generates an invisible mesh around each constellation, allowing users to click on them.

Snow Accumulation Shader

Unity
For my term project in Graphics 2, I created a surface shader for Unity to simulate snow accumulation. Snow in video games is usually made using either a pre-modelled snow mesh or a flat snow shader, resulting in snow that isn't affected by changes in weather. By allowing the depth and accumulation direction to vary depending on the weather, I created snow that is more dynamic and interesting than either of those methods.
More

Snow Accumulation Shader

The shader's options pane in the Unity editor.

My shader is implemented as a surface shader in Unity. The engine generates the fragment and vertex shaders based on the surface shader, and handles lighting, shadows, and tessellation. My code is responsible for everything related to snow, such as vertex deformation, textures, and fragment colour before shadows and lighting are calculated.
The shader uses a number of variables to define how the snow looks. Since Unity's shaders are applied to each material separately, a developer using the shader can control how snow accumulates on each mesh individually.

Left - Normal Scale set to 0; Right - Normal Scale set to 1

The 'Main Texture' and 'Normal Map' properties define how a mesh looks without snow. When a texture is provided the 'Base Color' property is used to tint the texture, otherwise it's used as the mesh colour. The normal map can be scaled with the 'Normal Scale' property, making the mesh appear bumpier or smoother. The normal map can be used to allow snow to accumulate in cracks or bumps on steep surfaces where it wouldn't appear otherwise.

Left - Normal Scale Under Snow set to 1; Right - Normal Scale Under Snow set to 0.2

The 'Normal Scale Under Snow' property adjusts the scale of the base normal map in areas that are covered by snow. This helps to simulate depth, by smoothing surface features as more snow accumulates.

Left - Using texture and snow normal map; Right - No texture or snow normal map

The 'Snow Texture' and 'Snow Normal Map' start to define how the mesh should look in areas covered by snow. The 'Snow Normal Scale' property adjusts the definition of the snow normal map, which is blended with the base mesh normal map so both normal maps are visible.

Left - Coverage set to 0.5, Direction (-0.35,1,0); Right - Coverage set to 1, Direction (1,1,0)

The 'Snow Direction' property is a vector that defines the direction that the snow is coming from, and allows the user to adjust the wind direction in a scene. This is used in coverage, depth, and blending calculations.

Coverage defines how much of a mesh is covered by snow. Coverage is based on the angle between the snow direction and the vertex normal.

Left - Offset set to 0; Right - Offset set to 0.15

The 'Offset' property offsets the Coverage cutoff for the snow in the vertex and surface shaders, causing the snow colour to extend past the deformed vertices. This helps to hide stretching and colour differences caused by deep snow.

Left - Colour blending set to 0; Middle - Colour blending set to 0.04; Right - Colour blending set to 0.44

The 'Color Blend' property blends the colour of the snow with the base colour, and also depends on the angle between the vertex normal and the snow direction. When set to a very small value, it helps to soften the edge of the snow; when set to larger values, it can be used to simulate a light snow.

Left - Depth set to 0; Middle - Depth set to 4; Right - Depth set to 4, with blending and tessellation

The 'Snow Depth' property, along with the 'Snow Depth Scale', adjusts how deep the snow is. This causes any vertices that are covered by snow (defined by the 'Coverage' property) to be moved towards the direction that the snow is coming from. When used without blending, tessellation, and offset, this can cause the mesh to look stretched and to have spikes in colour along the edge of the snow.

Left - Depth blending set to 0; Right - Depth blending set to 0.85

The 'Depth Blend' property adjusts how much the depth transformation is affected by the angle between the vertex normal and the snow direction. It helps to make the snow appear rounder and more natural.

Left - Tessellation Edge Length set to 50; Right - Tessellation Edge Length set to 2

The 'Tessellation Edge Length' variable is used by Unity for edge length based tessellation. This greatly reduces the stretching effect caused by snow depth, and allows the maximum depth map (described in the next section) to work on low poly meshes. Tessellation has a severe performance cost, so it should only be used when needed.

Left - No Depth Map; Middle - With Depth Map; Right - The Depth Map used

The 'Maximum Depth Map' property allows a developer to add a grayscale image defining how deep the snow can get at any point on the mesh. This works by multiplying the depth by the gray value when moving each vertex. When trying to increase the snow depth on low poly meshes, entire faces of the mesh will appear to become detached, caused by large differences between the normals on each face. When used in combination with tessellation, a maximum depth map will stop faces from detaching by preventing vertices near the edge of a face from moving. This could also be used on a terrain mesh to simulate snow drifting.

Left - Snow Occlusion map used to prevent snow accumulating directly beneath roof

The 'Snow Occlusion Map' property is a black and white texture used to prevent snow from accumulating in some areas. This can be used to stop snow from appearing inside a building or under an overhang.

Bejeweller

Java
The Bejeweller is a Java Swing application that plays Bejewelled 3. It interacts with the game in the same way a human player would – it scans the screen, detects the colour of each piece on the game board, and uses the mouse to perform a move. It can play a number of game modes effectively, and has exceeded my highest scores by far.
More

Bejeweller

I wrote the Bejeweller as a side project during my second year at the University of Manitoba. It uses Java's Robot class to detect the colour of each piece on the game board, finds a move to make a match, then uses the Robot class to use the cursor and perform the move.

In Lightning Mode (shown in the video), my highest score is 312 500 points, and the program's highest score is 10 628 850!
